遇到的问题

1. msg: blk is not a function. (In ‘blk(0, “”, documentName)’, ‘blk’ is false),

判断block是否为空时，不要用 if (blk != null) ，应该使用if (blk)。
因为当block为空时，if (blk != null)时，不知怎么的是YES，执行了下面的blk(0, “”, documentName);，可能和JS有关吧。

2. TypeError: cb.apply is not a function？

(1)block的参数格式写的有问题，比如：block从JS 传到OC时，需要用block包装一下，
block(“id”,blk)

基础

1. for循环相关

在for-in 与 经典for循环选择时，强烈建议使用 经典for循环:
for(var i=0;i<arry.count();i++)

介绍一下for-in：

  1. 当array.count()==0时，js 还会执行一下for的函数体，因此需要在for-in 前需要加上 if（array.count() > 0）{for-in}
  2. for var i in array{}其中的 i 代表 索引值！
  3. bug描述 :使用for in 时，遍历的array的item是比较复杂的string，

2. NSArray 注意

  1. 数组长度：
    JS 方式：array.toJS().length
    OC 方式：array.count()
  2. 获取数组元素：
    JS 方式：array[i]
    OC 方式：array.objectAtIndex(i);//array[i]失效
  3. NSArray / NSString / NSDictionary 不会自动转成对应的JS类型，怎么在.js中使用它们？（但是 JS类型会自动转换为OC类型，尽情的调用oc的set() 就行）
    (1)通过toJS()方法 转成对应的JS类型（可以用js的精简方法操作）
    (2)像普通 NSObject 一样使用它们 (可以熟练的使用OC的方法进行操作)

JSPatch Convertor ’s 不足：

  1. “_”翻译不到位：
  1. 在oc中的需要 [self class]用来调用加方法，热修复时 翻译成 self.class() ，JSPatch 不识别，需要 把[self class] 换成 类名
  2. block中含有if else时，经常漏掉 if else
  3. NSString.stringWithFormat(“%@”, array.count()) 会 把 “%@”漏成 “%”

应该注意

JSPatch 也会 hook 住对象的 forwradInvocation 方法，并且 swizzling 相应的 method ，使其指向转发 IMP ,由于 aspects 也是基于这两者实现的，使用时在64机器有可能会有问题，需要注意一下。
